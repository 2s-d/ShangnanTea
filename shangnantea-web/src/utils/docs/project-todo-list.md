# 商南茶文化平台 - 项目重构优化待办事项

## 术语说明

为避免混淆，特此说明本文档中使用的关键术语：

- **信息提示**：指项目全局的三层提示架构（API层/业务层/UI层），用于显示操作反馈、状态通知等UI提示内容。
- **消息模块**：特指六大业务模块之一的功能模块，负责处理系统通知、私信、关注收藏、个人主页等业务功能。

## 1. 信息提示系统重构

### 1.1 Message.vue 组件更新计划

**背景**：
- 项目已实施了基于三层架构（API层/业务层/UI层）的新信息提示系统
- 创建了六大业务模块的信息提示工具（userMessages、teaMessages等）
- 当前底层UI组件 Message.vue 未完全适配新架构需求

**问题**：
- Message.vue 不支持信息提示队列管理，导致多条提示可能相互覆盖
- 缺乏信息提示来源标识，不利于开发调试
- 无法区分不同模块的信息提示样式和行为
- 与 MessageMonitor.vue 的集成不够紧密

**优化目标**：
- 增强 Message.vue 支持信息提示队列显示
- 添加信息提示身份识别（来源模块、层级）
- 优化信息提示动画和交互体验
- 完善与监控组件的配合

**实施建议**：
- 保留现有API的同时扩展新功能
- 添加信息提示堆叠显示机制
- 实现信息提示优先级排序
- 增加可配置选项（位置、持续时间等）
- 确保与六大模块（包括消息模块）的信息提示工具良好兼容

## 实施计划

- **优先级**：高（影响用户体验和开发效率）
