# 订单状态与支付状态系统分析

## 一、当前状态定义

### 1.1 订单状态（Order Status）

| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| 0 | 待付款 | 订单已创建，等待用户支付 |
| 1 | 待发货 | 订单已支付，等待商家发货 |
| 2 | 待收货 | 订单已发货，等待用户确认收货 |
| 3 | 已完成 | 订单已完成（用户确认收货或自动确认） |
| 4 | 已取消 | 订单被取消（用户取消或超时取消） |
| 5 | 退款中 | 用户申请退款，等待商家/管理员审核 |
| 6 | 已退款 | 退款申请已通过，退款完成 |

### 1.2 支付状态（Payment Status）

| 状态码 | 状态名称 | 说明 |
|--------|---------|------|
| 0 | 待支付 | 支付单已创建，等待支付 |
| 1 | 支付成功 | 支付已完成 |
| 2 | 支付失败 | 支付失败（如余额不足、网络错误等） |
| 3 | 已关闭 | 支付单已关闭（超时、取消等） |

## 二、状态流转图

### 2.1 正常流程

```
创建订单
  ↓
订单状态=0（待付款），支付单状态=0（待支付）
  ↓
用户选择支付方式并支付
  ↓
支付成功回调
  ↓
订单状态=1（待发货），支付单状态=1（支付成功）
  ↓
商家发货
  ↓
订单状态=2（待收货）
  ↓
用户确认收货 / 7天后自动确认
  ↓
订单状态=3（已完成）
```

### 2.2 取消流程

```
订单状态=0（待付款）
  ↓
用户取消 / 30分钟超时自动取消
  ↓
订单状态=4（已取消），支付单状态=3（已关闭）
```

```
订单状态=1（待发货）
  ↓
用户取消订单
  ↓
订单状态=4（已取消），恢复库存
```

### 2.3 退款流程

```
订单状态=1（待发货）/ 2（待收货）/ 3（已完成）
  ↓
用户申请退款（7天内）
  ↓
订单状态=5（退款中），记录refundBeforeStatus
  ↓
商家/管理员审核
  ├─ 同意退款
  │   ↓
  │   订单状态=6（已退款），恢复库存和销量
  │
  └─ 拒绝退款
      ↓
      订单状态恢复为refundBeforeStatus
```

### 2.4 支付失败流程

```
支付单状态=0（待支付）
  ↓
用户发起支付但失败
  ↓
支付单状态=2（支付失败）
  ↓
订单状态仍为0（待付款），用户可以重新支付
```

## 三、超时与自动处理规则

### 3.1 支付超时

- **规则**：订单创建后30分钟内未支付，自动取消订单
- **处理**：
  - 订单状态：0（待付款）→ 4（已取消）
  - 支付单状态：0（待支付）→ 3（已关闭）
  - 恢复库存（如果已扣减）

### 3.2 自动确认收货

- **规则**：订单发货后7天，用户未确认收货，系统自动确认
- **处理**：
  - 订单状态：2（待收货）→ 3（已完成）
  - 更新完成时间

### 3.3 退款申请时效

- **规则**：订单完成后7天内可以申请退款，超过7天不允许申请
- **处理**：
  - 检查订单完成时间
  - 如果超过7天，返回错误码

## 四、不同角色查看订单详情的区别

### 4.1 用户角色（User）

**可见信息：**
- 订单基本信息（订单号、下单时间、订单状态）
- 收货信息（收货人、电话、地址）
- 商品信息（商品名称、规格、数量、价格）
- 金额信息（商品金额、运费、实付金额）
- 物流信息（待收货/已完成状态可见）
- 支付信息（支付方式、支付时间）
- 退款信息（如果申请了退款）

**可执行操作：**
- 待付款：立即支付、取消订单
- 待发货：修改收货地址
- 待收货：查看物流、确认收货
- 已完成：待评价/已评价、查看物流、申请退款（7天内）
- 退款中：查看退款进度
- 已退款：查看退款详情

### 4.2 商户角色（Shop）

**可见信息：**
- 订单基本信息（订单号、下单时间、订单状态）
- 买家信息（买家ID）
- 收货信息（收货人、电话、地址）
- 商品信息（商品名称、规格、数量、价格）
- 金额信息（商品金额、运费、实付金额）
- 物流信息（已发货/待收货/已完成状态可见）
- 支付信息（支付方式、支付时间）
- 退款信息（如果申请了退款）

**可执行操作：**
- 待发货：发货
- 已发货/待收货/已完成：查看物流
- 退款中：处理退款（同意/拒绝）

### 4.3 管理员角色（Admin）

**可见信息：**
- 订单基本信息（订单号、下单时间、订单状态）
- 买家信息（买家ID）
- 收货信息（收货人、电话、地址）
- 商品信息（商品名称、规格、数量、价格）
- 金额信息（商品金额、运费、实付金额）
- 物流信息（所有状态可见）
- 支付信息（支付方式、支付时间）
- 退款信息（如果申请了退款）

**可执行操作：**
- 待发货：发货
- 已发货/待收货/已完成：查看物流
- 退款中：处理退款（同意/拒绝）
- 所有状态：查看完整订单信息（用于客服处理）

## 五、订单详情页数据完整性检查

### 5.1 当前缺失的字段

1. **退款信息**：
   - `refundStatus`：退款状态（0无退款申请,1申请中,2已同意,3已拒绝）
   - `refundReason`：用户退款原因
   - `refundRejectReason`：商家/管理员拒绝原因
   - `refundApplyTime`：退款申请时间
   - `refundProcessTime`：退款处理时间

2. **支付状态信息**：
   - `paymentStatus`：支付单状态（0待支付,1成功,2失败,3关闭）
   - `paymentId`：支付单号

3. **时间字段**：
   - `cancelTime`：取消时间
   - `cancelReason`：取消原因

### 5.2 需要补充的字段

在 `OrderDetailVO` 中需要添加：
- 退款相关字段（或通过单独的退款详情接口获取）
- 支付状态字段
- 取消相关字段

   - 返回完整的订单详情

### 6.2 前端修改

1. **OrderDetailPage.vue**：
   - 根据 `refundStatus` 显示退款信息
   - 根据 `paymentStatus` 显示支付状态
   - 根据角色显示不同的操作按钮和信息

2. **条件渲染**：
   - 用户角色：显示买家操作按钮
   - 商户/管理员角色：显示卖家操作按钮
   - 根据订单状态和支付状态显示不同的提示信息

### 6.3 超时处理（定时任务）

需要实现定时任务：
1. **支付超时检查**：每5分钟检查一次，将超过30分钟未支付的订单自动取消
2. **自动确认收货**：每天检查一次，将发货超过7天的订单自动确认收货

## 七、状态闭环验证

### 7.1 状态转换合法性检查

| 当前状态 | 允许转换到的状态 | 不允许的状态 |
|---------|----------------|-------------|
| 0（待付款） | 1（待发货）、4（已取消） | 2、3、5、6 |
| 1（待发货） | 2（待收货）、4（已取消）、5（退款中） | 0、3、6 |
| 2（待收货） | 3（已完成）、5（退款中） | 0、1、4、6 |
| 3（已完成） | 5（退款中） | 0、1、2、4、6 |
| 5（退款中） | 6（已退款）、原状态（拒绝退款） | 0、1、2、3、4 |

### 7.2 支付状态与订单状态的一致性

- 订单状态=0（待付款）时，支付单状态应为0（待支付）或3（已关闭）
- 订单状态=1（待发货）时，支付单状态应为1（支付成功）
- 订单状态=4（已取消）且未支付时，支付单状态应为3（已关闭）
- 订单状态=6（已退款）时，支付单状态应为1（支付成功），但需要记录退款信息
