## 四、开发规范提醒

### 4.1 数据流向规范

```
组件 → Vuex Actions → API → Mock/后端 → Vuex Mutations → State → 组件
```

- 所有数据操作必须通过Vuex
- 组件不直接调用API
- API层负责与后端通信

### 4.2 文件命名规范

- API方法：动词+名词（如 `getTeaList`）
- Vuex Actions：动词开头（如 `fetchTeaList`）
- Vuex Mutations：全大写下划线（如 `SET_TEA_LIST`）

### 4.3 测试规范

每个功能完成后必须测试：
1. 正常流程测试
2. 异常流程测试
3. 边界条件测试
4. UI交互测试
